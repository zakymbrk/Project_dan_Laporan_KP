USE CASE DESCRIPTION:

1. UC-01: Login Sistem
   - Aktor: Admin/Hubin, Siswa
   - Deskripsi: Pengguna melakukan autentikasi untuk mengakses sistem

2. UC-02: Kelola Data Siswa
   - Aktor: Admin/Hubin
   - Deskripsi: Mengelola data siswa (tambah, edit, hapus, import)

3. UC-03: Ajukan PKL
   - Aktor: Siswa
   - Deskripsi: Siswa membuat pengajuan PKL ke DUDI tertentu

4. UC-04: Verifikasi Pengajuan
   - Aktor: Admin/Hubin
   - Deskripsi: Memverifikasi dan menyetujui/tolak pengajuan PKL

5. UC-05: Lihat Status Pengajuan
   - Aktor: Siswa
   - Deskripsi: Siswa melihat status pengajuan PKL mereka

6. UC-06: Kelola DUDI
   - Aktor: Admin/Hubin
   - Deskripsi: Mengelola data Dunia Usaha/Dunia Industri

7. UC-07: Kelola Pembimbing
   - Aktor: Admin/Hubin
   - Deskripsi: Mengelola data pembimbing dan assign ke siswa

8. UC-08: Kelola Pengumuman
   - Aktor: Admin/Hubin
   - Deskripsi: Membuat dan mengelola pengumuman untuk siswa

9. UC-09: Lihat Pengumuman
   - Aktor: Siswa
   - Deskripsi: Siswa melihat pengumuman yang dipublikasikan

==================================================
6. FLOW DATA SISTEM
==================================================

LOGIN FLOW:
1. User mengakses halaman login
2. Sistem memvalidasi username & password
3. Jika valid, buat session
4. Redirect ke dashboard sesuai role
5. Jika tidak valid, tampilkan error

PENGAJUAN PKL FLOW:
1. Siswa login dan akses menu pengajuan
2. Siswa memilih DUDI dan mengisi form
3. Siswa upload berkas pendukung
4. Siswa kirim pengajuan
5. Sistem simpan ke database dengan status "pending"
6. Admin/Hubin menerima notifikasi
7. Admin/Hubin verifikasi pengajuan
8. Admin/Hubin update status (diterima/ditolak)
9. Sistem kirim notifikasi ke siswa
10. Siswa dapat melihat status terbaru

VERIFIKASI PENGAJUAN FLOW:
1. Admin/Hubin login dan akses menu pengajuan
2. Admin/Hubin lihat daftar pengajuan
3. Admin/Hubin pilih pengajuan untuk diverifikasi
4. Admin/Hubin review berkas dan data
5. Admin/Hubin tentukan keputusan (terima/tolak)
6. Admin/Hubin input catatan/alasan
7. Sistem update status dan kirim notifikasi
8. Log aktivitas disimpan untuk audit trail

PENGUMUMAN FLOW:
1. Admin/Hubin login dan akses menu pengumuman
2. Admin/Hubin buat pengumuman baru
3. Admin/Hubin isi judul, isi, dan tanggal
4. Admin/Hubin publikasikan pengumuman
5. Sistem simpan ke database
6. Siswa dapat melihat pengumuman di dashboard
